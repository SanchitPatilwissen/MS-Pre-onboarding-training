1) S - Single Responsibility Principle --> A class should have single Responsibility.Avoids irrelevant modifications to unrelated code.

2) O - Open closed Principle --> Donot modify the previous code of the classs. Extend the class and make changes. Avoids irrelevant modifications to unrelated code.

3) L - Liskov's Substitution Principle --> Derived or child classes must be substitutable for their base or parent classes.

4) I - Interface Segregation Principle --> Donot force any client to implement an interface which is irrelevant to them

5) D - Dependency Inversion Principle --> Importance of Abstractions. High Level modules should not depend on low level modules.
